## Best Practices

1. **Keep JSON clean:** Use consistent indentation (2 or 4 spaces)
2. **Validate early:** Always validate JSON before using it in production
3. **Use meaningful keys:** Choose descriptive names for object properties
4. **Avoid redundancy:** Don't repeat data unnecessarily
5. **Document structure:** Use comments in your code explaining JSON schemas

## File Structure Tips

- Use **objects** for key-value pairs (entities, configurations)
- Use **arrays** for collections of similar items
- Keep nesting levels reasonable (avoid excessive depth)
- Use lowercase with underscores for key names (snake_case)
